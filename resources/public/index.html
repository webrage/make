Hello world

3 September 2018

I have been wanting to play with Cryogen for a while now. It is a clojure based helper for creating static web sites.

Having had a quick look at it I decided to go with the lotus theme by KingMob. It has a simple and uncluttered look that I like.

setup issues

With any new process there is stuff to learn and figure out.

cryogen

Being familiar with clojure I was quickly able to bring the getting started website to life at localhost. I use the Atom IDE for occasional clojure programming, so Cryogen and Atom were a good fit for me.

The development process involves running a ring server from the project directory. This enables Cryogen to generate a new version of the website (in the IDE, and in the browser) every time the file that you are working on is saved. Nice.

lotus theme

The fun began when I decided to go with the lotus theme and the website wouldn't render in the browser. This theme uses SASS for its style sheets, so I had to install the SASS command-line app and update the config file. I was happy to install this as it looks interesting. Still no luck. Turns out the latest version of the SASS command-line app doesn't work the same as previously, leaving Cryogen throwing an error when it called SASS. My current work-around is to manually call SASS to parse the theme scss files into css files (into the same directory). This is only needed after a scss file is updated, which shouldn't be too often. This work around will do until Cryogen and SASS are back in sync.

github

The next challenge was to store the website on github. Not hard, just that I hadn't done it before. Turns out Atom, the IDE I use, makes the process pretty easy once you get the settings right. I just wanted to save the bare bones website to github without any extraneous file / folder clutter. It took a bit of playing around, but not too long.

netlify

I thought it would be good to give netlify a go. It offers free hosting for static web sites. Netlify can grab any updates to the github website repository and update the website from there. The most important change I had to make was to ensure there was no blog prefix in the Cryogen config file, otherwise the relative paths in the website were wrong.

the final process

To create a new post on my website, the steps are as follows:

  1. Fire up the Atom IDE and open up the cryogen website code
  2. If I change one of the scss style sheet files, run a script to create associated css files
  3. Run a ring server so I can check the changes as I save them
  4. Use markdown to write a post or page, saving occasionally to check it looks OK
  5. In Atom, push the committed changes to github.
  6. Wait a minute or two and check the website is all OK.
